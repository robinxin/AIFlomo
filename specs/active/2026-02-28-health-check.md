# Feature Spec: 健康检查接口

**作者**: AIFlomo Team
**日期**: 2026-02-28
**状态**: 已批准

---

## 1. 背景

需要一个标准健康检查接口，供运维监控和部署脚本判断服务是否正常运行。

## 2. 用户故事

```
作为运维工程师
我希望调用一个 HTTP 接口就能判断服务是否存活
以便在服务异常时及时告警或重启
```

## 3. 功能描述

- `GET /api/health` 返回服务状态和当前时间戳
- 无需认证
- 响应时间 < 200ms

## 4. 数据模型

无需数据库操作。

## 5. API 设计

**请求**
```
GET /api/health
```

**响应 200**
```json
{
  "status": "ok",
  "timestamp": "2026-02-28T09:00:00.000Z"
}
```

**错误响应**

服务不可用时返回 503，body 同上但 `status` 为 `"error"`。

## 6. 边界条件

- 接口本身不应因业务逻辑报错（不查数据库、不验证 session）
- `timestamp` 使用 ISO 8601 格式

## 7. 不包含

- 数据库连接状态检查
- 依赖服务（Redis、外部 API）的探活

## 8. 验证标准

- `GET /api/health` 返回 200
- 响应体包含 `status: "ok"` 和合法的 `timestamp` 字段
- 测试覆盖：正常返回、timestamp 格式校验
