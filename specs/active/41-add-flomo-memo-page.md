# 功能规格：Flomo 笔记页面

**Feature Branch**: `feat/41-flomo-memo-page`
**创建日期**: 2026-03-10
**状态**: 草稿
**关联 Issue**: #41

## Overview

仿 Flomo 风格的 Web 单页笔记应用。用户登录后进入双栏主界面，可快速记录想法、
添加标签/图片/链接，并通过侧边栏过滤、搜索、热力图等功能管理笔记。
Pro 功能入口（微信输入、每日回顾、AI 洞察、随机漫步）均以"购买 Pro 会员"浮窗拦截。

---

## Target Users

个人用户，希望以最低摩擦记录零散想法，并通过标签和时间线回顾内容。

### Section 1 · 左侧边栏

#### 1.1 用户信息区

```
┌──────────────────────┐
│  金兰  [PRO]         │   ← 昵称 + PRO 徽章（若已订阅）
│  [头像/首字母圆形]   │
│  [退出登录] (下拉)   │
└──────────────────────┘
```

- 昵称取登录用户的 `nickname`
- 点击昵称/头像展开下拉菜单，包含「退出登录」选项
- 退出登录：清除 session token → 跳转 `/login`

#### 1.2 统计栏

```
  6          1          10
 笔记        标签         天
```

三列水平排布，数字加粗大字，标签居中。

| 字段 | 含义 |
|------|------|
| 笔记 | 当前用户全部未删除笔记总数 |
| 标签 | 当前用户至少有 1 条笔记的不同标签数量 |
| 天   | 用户创建过笔记的不同日期总数（连续或非连续均计入） |

#### 1.3 热力图

- 显示最近 ~16 周（约 4 个月）的日历格，每格代表一天
- 颜色深浅映射当日新建笔记数：
  - 0 条 → `#2a2a2a`（最暗，几乎不可见）
  - 1–2 条 → `#2d4a3a`
  - 3–5 条 → `#2d5a3d`（品牌绿，中等）
  - 6+ 条 → `#3d7a55`（最亮）
- 横轴月份标签（十二月、一月、二月、三月…）显示在格子下方
- 悬浮某格时 tooltip 显示：`YYYY-MM-DD · N 条笔记`

#### 1.4 导航菜单

```
■  全部笔记         ← 高亮选中（绿色背景）
     无标签
     有图片
     有链接
     有语音
💬  微信输入         ← Pro 入口
✦   每日回顾         ← Pro 入口
⊙   AI 洞察          ← Pro 入口
✿   随机漫步         ← Pro 入口
```

**笔记类型子项**（在「全部笔记」下缩进展示）

| 子项 | 过滤条件 |
|------|---------|
| 无标签 | `note.tags` 为空数组 |
| 有图片 | `note.images.length > 0` |
| 有链接 | `note.content` 中包含 URL |
| 有语音 | `note.audioUrl` 不为空 |

点击任意子项，笔记流只显示符合该条件的笔记；再次点击「全部笔记」恢复全量。

**Pro 入口行为**：点击「微信输入」「每日回顾」「AI 洞察」「随机漫步」中任意一项，
弹出 Pro 会员购买浮窗（见 Section 4），不跳转页面。

#### 1.5 全部标签

标题行「全部标签」（灰色小字）

每行格式：
```
#  科幻    3
#  工作    12
```

- `#` 图标 + 标签名 + 右侧笔记数量（该标签下未删除笔记总数）
- 按笔记数量降序排列
- 点击某标签行，笔记流过滤为该标签的笔记
- 当某标签的笔记数降为 0 时，该行自动消失

#### 1.6 回收站

```
🗑  回收站   (5)
```

- 括号内显示已删除笔记总数；0 条时不显示数字
- 点击跳转「回收站」视图（见 Section 5）

### Section 2 · 新建笔记输入框

固定在笔记流顶部，始终可见（不随笔记流滚动）。

```
┌─────────────────────────────────────────────────┐
│  现在的想法是...                                  │
│                                                   │
│  #  🖼  |  Aa  ≡  ≡  |  @                  [►]  │
└─────────────────────────────────────────────────┘
```

#### 工具栏按钮

| 按钮 | 功能 |
|------|------|
| `#`  | 标签输入（见下方「标签输入」规则） |
| `🖼` | 上传图片附件（最多 9 张） |
| `Aa` | 切换富文本工具栏（粗体/下划线/高亮） |
| `≡`（有序） | 插入有序列表 |
| `≡`（无序） | 插入无序列表 |
| `@`  | @引用其他笔记（弹出笔记搜索下拉） |
| `►`  | 提交笔记 |

#### 标签输入

- 点击 `#` 按钮 或在正文中输入 `#` 字符，弹出标签自动补全下拉
- 下拉列表显示已有标签（模糊匹配），最多展示 8 条
- 选择或回车确认：在正文中插入 `#tagname` chip，同时将该标签记录到 `note.tags`
- 可输入新标签名并回车，创建新标签
- 标签名大小写不敏感，存储时统一小写，最长 32 字符

#### 链接自动识别

- 正文中输入 URL（`http://` 或 `https://` 开头）时，提交后自动提取到 `note.links[]`
- 笔记卡片中以蓝色可点击文本渲染链接

#### 图片上传

- 点击 `🖼` 打开系统文件选择器，支持 jpg/png/gif/webp，单张最大 10MB
- 选中后在输入框内显示缩略图预览
- 提交时调用 `POST /api/upload/image` 上传，写入 `note.images[]`

#### 提交规则

- 正文不能为空（纯空格/换行视为空）
- 提交成功：新笔记卡片**乐观地**插入到笔记流顶部，输入框清空
- 提交失败：回滚乐观插入，顶部 toast 提示「保存失败，请重试」

### Section 5 · 回收站视图

路由：`/trash`（或侧边栏内的子视图，无需整页跳转）

```
┌──────────────────────────────────────────────┐
│  回收站（5 条）              [清空回收站]     │
├──────────────────────────────────────────────┤
│  [笔记卡片]  删除于 2026-03-01               │
│     [恢复]  [彻底删除]                       │
│                                              │
│  [笔记卡片]  删除于 2026-02-28               │
│     [恢复]  [彻底删除]                       │
└──────────────────────────────────────────────┘
```

- 按 `deletedAt` 降序排列
- **恢复**：将笔记 `deleted` 置为 `false`，从回收站移除，重新出现在主笔记流
- **彻底删除**：弹出确认对话框 → 确认后永久删除记录及附件文件
- **清空回收站**：弹出确认对话框 → 确认后批量永久删除所有已删除笔记
- 回收站中的笔记不出现在主笔记流和搜索结果中

---

### Section 6 · 搜索

**触发方式**：点击顶栏搜索框，或按 `⌘K` / `Ctrl+K`

**搜索框位置**：主界面顶部导航栏右侧，常驻显示占位文字「⌘K」

**行为**

1. 输入关键词（防抖 200ms），笔记流替换为搜索结果
2. 支持 `#tagname` 语法：输入以 `#` 开头时按标签过滤
3. 匹配范围：笔记正文（全文）
4. 结果中匹配片段高亮（黄色背景）
5. 清空搜索词 → 恢复完整笔记流
6. 结果为空时显示「未找到相关笔记」

## Data Models

```
User {
  id:           string    (UUID)
  nickname:     string
  account:      string    (phone or email, unique)
  passwordHash: string
  createdAt:    timestamp
}

Note {
  id:          string    (UUID)
  userId:      string
  content:     string    (HTML)
  tags:        string[]
  images:      string[]  (file paths)
  audioUrl:    string?
  links:       string[]  (auto-extracted URLs)
  refs:        string[]  (referenced note IDs via @)
  deleted:     boolean   (default false)
  deletedAt:   timestamp?
  createdAt:   timestamp
  updatedAt:   timestamp
}
```

## API Endpoints

### Auth
| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/login` | 登录，返回 JWT（httpOnly cookie） |
| POST | `/api/auth/logout` | 登出，清除 cookie |

### Notes
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/notes` | 获取笔记列表；支持 `?tag=`、`?type=untagged\|image\|link\|audio`、`?q=`、`?page=`、`?limit=30` |
| POST | `/api/notes` | 创建笔记 |
| PATCH | `/api/notes/:id` | 更新笔记内容/标签 |
| DELETE | `/api/notes/:id` | 软删除（移入回收站） |
| GET | `/api/notes/trash` | 获取回收站列表 |
| POST | `/api/notes/:id/restore` | 从回收站恢复 |
| DELETE | `/api/notes/:id/permanent` | 永久删除单条 |
| DELETE | `/api/notes/trash` | 清空回收站 |

### Tags
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/tags` | 返回所有标签及各标签的笔记数量 |

### Stats
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/stats` | 返回 `{ totalNotes, totalTags, activeDays, heatmap }` |

`heatmap` 字段格式：`{ "2026-02-26": 3, "2026-02-27": 1, ... }`（最近 112 天）

### Uploads
| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/upload/image` | 上传图片，返回访问 URL |
| POST | `/api/upload/audio` | 上传音频，返回访问 URL |

## Non-Functional Requirements

- **Stack**：Node.js + Express 后端；Vite + Vanilla JS 前端（无重型框架）
- **Auth**：JWT 存 `httpOnly` cookie，有效期 7 天
- **Storage**：SQLite（better-sqlite3）+ 本地 `uploads/` 目录
- **Search**：FTS5 虚拟表，不使用 `LIKE '%q%'`
- **Performance**：10,000 条笔记场景下列表渲染 ≤ 200ms；搜索响应 ≤ 300ms（含 200ms 防抖）
- **Pagination**：每页 30 条，前端无限滚动（距底部 200px 时加载下一页）
- **Responsive**：桌面优先；< 768px 时侧边栏收起为抽屉（汉堡菜单触发）
- **Theme**：纯深色模式，无浅色切换s
