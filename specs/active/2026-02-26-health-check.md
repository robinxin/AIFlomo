# Feature Spec: 健康检查接口

**作者**: 方世研  
**日期**: 2026-02-26  
**状态**: 已批准  
**关联 Issue**: #2

---

## 1. 背景
需要一个极简的健康检查接口，用于验证服务是否正常运行。

## 2. 用户故事
```
作为运维或开发者，
我希望有一个无需鉴权的健康检查接口，
以便我可以快速确认服务状态。
```

## 3. 功能描述
- 提供一个 GET 接口用于健康检查
- 不依赖数据库或其他外部服务
- 返回结构固定、可用于自动化探测

## 4. 数据模型
- 无需新增数据表或字段

## 5. API 设计
- **GET** `/api/health`
- **响应** `200 OK`
```json
{
  "status": "ok"
}
```
- **错误码**：无

## 6. 边界条件
- 该接口必须在应用启动后可用
- 任何请求参数应被忽略

## 7. 不包含
- ❌ 鉴权
- ❌ 数据库存取
- ❌ 复杂诊断信息

## 8. 验证标准
- [ ] 请求 `/api/health` 返回 HTTP 200
- [ ] 返回 JSON 中包含 `status: "ok"`
