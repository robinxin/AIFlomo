## 用户认证

### UI 测试场景

#### 正常场景

- 用户在登录页输入有效邮箱和密码，成功登录并跳转到笔记主页
  - 前置条件：用户已注册账号，停留在 /login 页面
  - 操作步骤：
    1. 输入框填入邮箱 `test@example.com`
    2. 输入框填入密码 `12345678`
    3. 点击"登录"按钮
  - 预期结果：
    - 页面跳转至 /memo（笔记主页）
    - 显示成功提示："登录成功"
    - Session Cookie 已设置

- 用户在注册页输入有效信息，成功注册并自动登录
  - 前置条件：停留在 /register 页面
  - 操作步骤：
    1. 输入邮箱 `newuser@example.com`
    2. 输入昵称 `测试用户`
    3. 输入密码 `12345678`（≥ 8 字符）
    4. 点击"注册"按钮
  - 预期结果：
    - 页面跳转至 /memo
    - 显示成功提示："注册成功"
    - 右上角显示昵称"测试用户"

- 用户在笔记主页点击登出，返回登录页
  - 前置条件：用户已登录，停留在 /memo 页面
  - 操作步骤：
    1. 点击右上角用户头像/昵称
    2. 在下拉菜单中点击"登出"
  - 预期结果：
    - 页面跳转至 /login
    - Session Cookie 被清除
    - 显示提示："已登出"

#### 异常场景

- 未登录用户直接访问 /memo，自动跳转登录页
  - 前置条件：用户未登录（无有效 Session）

    1. 浏览器地址栏输入 `/memo`
    2. 按下回车

    - 显示提示："请先登录"

### API 测试场景

#### 正常场景

- 有效邮箱和密码注册新用户，返回 201
  - 操作步骤：
    1. 发送 POST /api/auth/register
    2. Body：`{ "email": "new@example.com", "password": "12345678", "nickname": "测试用户" }`
  - 预期结果：
    - 接口返回 201
    - 响应体：`{ data: { id, email, nickname }, message: "注册成功" }`
    - 数据库 users 表新增一条记录
    - Set-Cookie 响应头包含 Session ID

- 有效邮箱和密码登录，返回 200 并设置 Session
  - 操作步骤：
    1. 发送 POST /api/auth/login
    2. Body：`{ "email": "test@example.com", "password": "12345678" }`
  - 预期结果：
    - 接口返回 200
    - 响应体：`{ data: { id, email, nickname }, message: "登录成功" }`
    - Set-Cookie 响应头包含 Session ID

- 已登录用户调用登出接口，Session 被销毁
  - 操作步骤：
    1. 用户已登录（请求携带 Session Cookie）
    2. 发送 POST /api/auth/logout
  - 预期结果：
    - 接口返回 200
    - 响应体：`{ data: null, message: "已登出" }`
    - Set-Cookie 响应头清除 Session

- 已登录用户获取当前用户信息，返回 200
  - 操作步骤：
    1. 用户已登录（请求携带 Session Cookie）
    2. 发送 GET /api/auth/me
  - 预期结果：
    - 接口返回 200
    - 响应体：`{ data: { id, email, nickname }, message: "ok" }`

#### 异常场景

- 注册时邮箱格式错误，返回 400
  - 操作步骤：
    1. 发送 POST /api/auth/register
    2. Body：`{ "email": "notanemail", "password": "12345678", "nickname": "测试" }`
  - 预期结果：
    - 接口返回 400
    - 响应体：`{ data: null, error: "VALIDATION_ERROR", message: "请求参数不合法" }`

- 注册时密码长度不足 8 字符，返回 400
  - 操作步骤：
    1. 发送 POST /api/auth/register
    2. Body：`{ "email": "test@example.com", "password": "123", "nickname": "测试" }`
  - 预期结果：
    - 接口返回 400
    - 响应体：`{ data: null, error: "VALIDATION_ERROR", message: "请求参数不合法" }`

- 注册时邮箱已存在，返回 409
  - 操作步骤：
    1. 邮箱 `existing@example.com` 已注册
    2. 发送 POST /api/auth/register
    3. Body：`{ "email": "existing@example.com", "password": "12345678", "nickname": "测试" }`
  - 预期结果：
    - 接口返回 409
    - 响应体：`{ data: null, error: "EMAIL_EXISTS", message: "该邮箱已被注册" }`

- 登录时邮箱或密码错误，返回 401
  - 操作步骤：
    1. 发送 POST /api/auth/login
    2. Body：`{ "email": "test@example.com", "password": "wrongpassword" }`
  - 预期结果：
    - 接口返回 401
    - 响应体：`{ data: null, error: "INVALID_CREDENTIALS", message: "邮箱或密码错误" }`

- 未登录状态下访问 /api/auth/me，返回 401
  - 操作步骤：
    1. 不携带 Session Cookie
    2. 发送 GET /api/auth/me
  - 预期结果：
    - 接口返回 401
    - 响应体：`{ data: null, error: "Unauthorized", message: "请先登录" }`
