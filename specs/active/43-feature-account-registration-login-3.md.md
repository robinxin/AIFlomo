# 功能规格：账号注册与登录

**Feature Branch**: `feat/43-account-registration-login`
**创建日期**: 2026-03-11
**状态**: 草稿
**关联 Issue**: #43

## 用户场景与测试

### 用户故事 1 — 新用户注册（优先级：P1）

未注册用户访问注册页面，填写邮箱、昵称、密码并同意隐私协议后，成功创建账号并进入应用主界面，开始记录第一条 Memo。

**为什么是 P1**：这是用户进入 AIFlomo 生态的第一步，没有注册功能就无法建立用户体系，也无法实现多端同步、数据隔离等核心能力。

**独立测试方式**：访问注册页面，完成表单填写和提交，验证账号创建成功并能够使用该账号登录系统。

**验收场景**：

1. **假设** 用户首次访问 AIFlomo，**当** 用户在注册页面输入有效的邮箱、昵称（2-20 字符）、密码（8-20 字符）并勾选隐私协议后点击"注册"按钮，**则** 系统创建账号并自动登录，进入 Memo 列表页面。
2. **假设** 用户在注册页面，**当** 用户输入已被使用的邮箱并尝试注册，**则** 系统显示"该邮箱已被注册"提示，表单保持当前输入内容不清空。
3. **假设** 用户在注册页面，**当** 用户未勾选隐私协议就点击"注册"按钮，**则** 系统高亮隐私协议勾选框并显示"请阅读并同意隐私协议"提示，不提交表单。
4. **假设** 用户在注册页面，**当** 用户点击"返回登录"链接，**则** 系统跳转到登录页面，已输入的注册信息被清空。

---

### 用户故事 2 — 已有用户登录（优先级：P2）

已注册用户访问登录页面，输入邮箱和密码后成功登录，系统恢复用户的 Memo 数据和个人设置。

**为什么是 P2**：登录是用户重复使用产品的必经之路，虽然优先级略低于注册（因为必须先有账号才能登录），但它是用户留存的关键功能。

**独立测试方式**：使用已注册的账号在登录页面完成认证，验证能够访问该账号的历史数据和个性化配置。

**验收场景**：

1. **假设** 用户已拥有 AIFlomo 账号，**当** 用户在登录页面输入正确的邮箱和密码后点击"登录"按钮，**则** 系统验证通过并跳转到 Memo 列表页面，显示该用户的所有 Memo 记录。
2. **假设** 用户在登录页面，**当** 用户输入错误的邮箱或密码，**则** 系统显示"邮箱或密码错误，请重试"提示，密码输入框自动清空，邮箱输入框保持原内容。
3. **假设** 用户在登录页面，**当** 用户点击"立即注册"链接，**则** 系统跳转到注册页面，已输入的登录信息被清空。

---

### 用户故事 3 — 表单输入体验优化（优先级：P3）

用户在填写注册或登录表单时，系统实时反馈输入状态（合法性、错误提示），减少提交失败的挫败感。

**为什么是 P3**：这是提升用户体验的锦上添花功能，不影响核心注册/登录流程的可用性，但能显著降低用户操作摩擦。

**独立测试方式**：在注册/登录表单中故意输入不符合规则的内容（如无效邮箱、过短密码），验证实时错误提示是否及时准确。

**验收场景**：

1. **假设** 用户在注册页面输入邮箱时，**当** 用户输入不符合邮箱格式的字符串（如 "test@"）并失焦，**则** 邮箱输入框下方立即显示"请输入有效的邮箱地址"红色提示文字。
2. **假设** 用户在注册页面输入密码时，**当** 用户输入少于 8 个字符的密码并失焦，**则** 密码输入框下方立即显示"密码长度至少为 8 个字符"提示。
3. **假设** 用户在注册页面输入昵称时，**当** 用户输入超过 20 个字符，**则** 系统阻止继续输入并在输入框下方显示"昵称最多 20 个字符"提示。

---

### 边界场景

- **网络异常**：当用户提交注册/登录请求时网络中断，系统应显示"网络连接失败，请稍后重试"提示，保留已填写内容，允许用户重新提交。
- **并发注册**：当多个用户同时使用相同邮箱注册，系统应确保只有一个注册成功，其他用户收到"该邮箱已被注册"提示。
- **Session 过期**：当用户登录后长时间未操作导致会话过期，系统应在用户下次操作时提示"登录已过期，请重新登录"并跳转到登录页面。
- **密码重置（未实现）**：当用户忘记密码时，系统暂不提供密码重置功能（后续迭代考虑），登录失败提示中不显示"忘记密码"链接。

## 功能需求

### 功能性需求

- **FR-001**：系统必须允许用户使用邮箱、昵称、密码创建新账号。
- **FR-002**：系统必须验证邮箱格式符合标准邮箱规范（包含 @ 和域名）。
- **FR-003**：系统必须确保同一邮箱只能注册一次，重复注册时向用户明确提示。
- **FR-004**：系统必须要求用户同意隐私协议后才能完成注册，未勾选时阻止提交。
- **FR-005**：系统必须允许已注册用户使用邮箱和密码登录。
- **FR-006**：系统必须在用户登录成功后创建会话，保持用户登录状态直到主动登出或会话过期。
- **FR-007**：系统必须在用户输入错误的邮箱或密码时，明确提示"邮箱或密码错误"而不泄露具体是哪个字段错误（安全考虑）。
- **FR-008**：系统必须在注册成功后自动登录用户，无需二次输入密码。
- **FR-009**：系统必须提供"返回登录"和"立即注册"的快速切换入口，方便用户在两个页面间导航。
- **FR-010**：系统必须在表单提交过程中显示加载状态，防止用户重复提交。

### 核心数据实体

- **用户（User）**：代表 AIFlomo 的注册用户，包含唯一标识、邮箱（用于登录和识别）、昵称（用于显示）、密码（加密存储）、隐私协议同意时间戳、账号创建时间。
- **会话（Session）**：代表用户的登录状态，包含会话标识、关联的用户、创建时间、过期时间。「推断」会话有效期为 7 天，到期后需要重新登录。

## 界面与交互

### 页面/视图清单

| 视图名称 | 入口 | 功能职责 |
|---------|------|---------|
| 注册页面 | 首次访问 AIFlomo 或从登录页面点击"立即注册" | 收集用户注册信息（邮箱、昵称、密码、隐私协议同意），创建新账号 |
| 登录页面 | 用户访问 AIFlomo 或从注册页面点击"返回登录" | 验证用户身份（邮箱、密码），建立会话 |

### 核心操作流

**注册流程**：

1. 用户访问注册页面 → 看到邮箱、昵称、密码输入框、隐私协议勾选框、"注册"按钮、"返回登录"链接
2. 用户依次填写邮箱、昵称、密码 → 系统实时验证格式（失焦时显示错误提示）
3. 用户勾选隐私协议 → 勾选框状态变为已选中
4. 用户点击"注册"按钮 → 系统显示加载状态（按钮文字变为"注册中..."且禁用）
5. 系统验证通过 → 创建账号、自动登录、跳转到 Memo 列表页面
6. （可选）用户点击"返回登录" → 清空表单并跳转到登录页面

**登录流程**：

1. 用户访问登录页面 → 看到邮箱、密码输入框、"登录"按钮、"立即注册"链接
2. 用户依次填写邮箱、密码 → 无实时验证，仅在提交时验证
3. 用户点击"登录"按钮 → 系统显示加载状态（按钮文字变为"登录中..."且禁用）
4. 系统验证通过 → 创建会话、跳转到 Memo 列表页面
5. （可选）用户点击"立即注册" → 清空表单并跳转到注册页面

### 界面状态

| 状态 | 触发条件 | 用户看到什么 |
|------|---------|------------|
| 初始状态 | 页面加载完成 | 所有输入框为空，"注册"/"登录"按钮可点击，无错误提示 |
| 输入中状态 | 用户聚焦输入框 | 当前输入框高亮边框（蓝色），其他输入框恢复默认边框 |
| 验证错误状态 | 用户失焦输入框且内容不符合规则 | 输入框下方显示红色错误提示文字，输入框边框变为红色 |
| 提交加载状态 | 用户点击"注册"/"登录"按钮 | 按钮文字变为"注册中..."/"登录中..."，按钮禁用无法再次点击，所有输入框禁用 |
| 提交失败状态 | 后端返回错误（如邮箱已存在、密码错误） | 在表单顶部显示红色错误提示框，具体错误信息在框内，按钮恢复可点击状态，输入框恢复可编辑 |
| 提交成功状态 | 注册/登录成功 | 页面自动跳转到 Memo 列表，不停留在当前页面 |

### 关键交互规则

- **表单验证时机**：注册页面的邮箱、昵称、密码在失焦（blur）时触发格式验证，登录页面仅在提交时验证（减少登录摩擦）。
- **错误提示位置**：字段级错误（格式不正确）显示在对应输入框下方；服务端错误（邮箱已注册、密码错误）显示在表单顶部的统一提示区域。
- **隐私协议勾选**：必须主动勾选，系统不预设为已勾选，未勾选时提交表单会阻止并高亮提示该勾选框。
- **页面跳转清空规则**：从注册页跳转到登录页（或反向）时，清空所有输入内容，避免数据混淆。
- **密码显示/隐藏**：「推断」密码输入框右侧提供眼睛图标，点击可切换明文/密文显示，默认为密文。

## 成功标准

- **SC-001**：90% 的用户能在首次尝试时成功完成注册流程，无需查阅帮助文档或联系客服。
- **SC-002**：用户从打开注册页面到完成注册进入 Memo 列表，整个流程耗时不超过 2 分钟（包含阅读隐私协议时间）。
- **SC-003**：已注册用户从打开登录页面到进入 Memo 列表，整个流程耗时不超过 30 秒。
- **SC-004**：表单验证错误提示在用户失焦后立即显示（延迟不超过 200 毫秒），让用户感知到实时反馈。
- **SC-005**：注册/登录请求在正常网络环境下，从点击按钮到收到服务端响应（成功或失败）不超过 3 秒。
