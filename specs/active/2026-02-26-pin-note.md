# Feature Spec: 置顶笔记

**作者**: Claude
**日期**: 2026-02-26
**状态**: 草稿
**关联 Issue**: -

---

## 1. 背景

用户在持续记录笔记后，某些重要的笔记会被新笔记淹没。
用户需要一种简单的方式将重要笔记"钉"在列表顶部，方便随时查看。

## 2. 用户故事

```
作为一个有大量笔记的用户，
我希望可以将重要的笔记置顶，
以便在打开应用时第一眼就能看到关键内容，不会被新笔记覆盖。
```

## 3. 功能描述

### 3.1 核心交互流程

```
用户在笔记卡片上点击「更多」菜单
    ↓
菜单中出现「置顶」选项
    ↓
点击后笔记移动到列表顶部，显示置顶图标（📌）
    ↓
再次点击「更多」菜单，选项变为「取消置顶」
    ↓
点击后笔记回到正常时间排序位置
```

### 3.2 页面布局

```
┌─────────────────────────────────────┐
│  ┌─────────────────────────────┐   │
│  │ 📌 这是一条置顶笔记         │   │
│  │ #重要                       │   │
│  │                    10:30    │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │ 📌 另一条置顶笔记           │   │
│  │                    09:15    │   │
│  └─────────────────────────────┘   │
│  ─────── 2026年2月26日 ───────     │
│  ┌─────────────────────────────┐   │
│  │ 普通笔记按时间排序           │   │
│  │                    14:30    │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

### 3.3 详细需求

**置顶操作**：
- [ ] 在笔记卡片的「更多」菜单中增加「置顶 / 取消置顶」选项
- [ ] 点击后立即生效，无需确认
- [ ] 操作后列表自动重新排序

**排序规则**：
- [ ] 置顶笔记始终排在列表最上方
- [ ] 多条置顶笔记之间按置顶时间倒序（最近置顶的在最上）
- [ ] 非置顶笔记保持原有的按创建时间倒序排列

**视觉标识**：
- [ ] 置顶笔记左上角显示 📌 图标
- [ ] 置顶笔记与普通笔记之间无需额外分隔线

## 4. 数据模型

在 Note 模型中新增一个字段：

```prisma
model Note {
  // ...existing fields
  pinnedAt  DateTime?   // null 表示未置顶，有值表示置顶时间
}
```

## 5. API 设计

**切换置顶状态**
```
PATCH /api/v1/notes/:id/pin
```

请求体：
```json
{
  "pinned": true
}
```

成功响应：
```json
{
  "success": true,
  "data": {
    "id": "clxyz...",
    "pinnedAt": "2026-02-26T06:00:00.000Z"
  },
  "error": null
}
```

取消置顶（`pinned: false`）成功响应：
```json
{
  "success": true,
  "data": {
    "id": "clxyz...",
    "pinnedAt": null
  },
  "error": null
}
```

**笔记列表排序调整**

现有 `GET /api/v1/notes` 接口无需新增参数，排序逻辑改为：
```
ORDER BY pinnedAt IS NOT NULL DESC, pinnedAt DESC, createdAt DESC
```

## 6. 边界条件

- 已删除的笔记（在回收站中）不允许置顶
- 置顶笔记数量不做限制
- 置顶状态在编辑笔记时保持不变

## 7. 不包含

- ❌ 拖拽排序
- ❌ 置顶笔记分组/折叠
- ❌ 置顶数量上限

## 8. 验证标准

- [ ] 可以置顶和取消置顶笔记
- [ ] 置顶笔记始终显示在列表顶部
- [ ] 多条置顶笔记按置顶时间排序正确
- [ ] 回收站中的笔记不可置顶
- [ ] API 返回格式符合 Spec
