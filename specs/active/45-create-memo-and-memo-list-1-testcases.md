## 创建纯文本笔记

### UI 测试场景

#### 正常场景

- 输入有效内容并点击发送，笔记出现在列表顶部
  - 前置条件：用户已登录，停留在主界面首页（"全部笔记"分类）
  - 操作步骤：
    1. 点击页面底部输入框，键盘弹起
    2. 输入内容"今天学到的新概念"
    3. 点击"发布"按钮
  - 预期结果：
    - 输入框内容清空
    - 笔记列表顶部出现包含"今天学到的新概念"的笔记卡片
    - 显示成功提示（Toast："发布成功" 或 "创建成功"）
    - 笔记卡片显示创建时间

- 输入 10000 字符内容，发布成功
  - 前置条件：用户已登录，停留在主界面
  - 操作步骤：
    1. 在输入框中输入 10000 字符的文本
    2. 输入框底部字数统计显示"10000/10000"
    3. 点击"发布"按钮
  - 预期结果：
    - 发送成功
    - 笔记完整内容显示在列表顶部
    - 显示成功提示

- 创建笔记后下拉刷新，新笔记仍显示在列表顶部
  - 前置条件：用户已成功创建一条笔记
  - 操作步骤：
    1. 在笔记列表顶部下拉刷新
    2. 等待刷新完成
  - 预期结果：
    - 刚创建的笔记仍显示在列表最顶部
    - 列表顺序按创建时间倒序排列

#### 异常场景

- 输入框为空时点击发送，前端给出提示
  - 前置条件：用户已登录，输入框为空
  - 操作步骤：
    1. 不输入任何内容（输入框为空）
    2. 点击"发送"按钮
  - 预期结果：
    - 发送请求未发出（或接口返回 400）
    - 界面显示错误提示："内容不能为空"
    - 输入框保持聚焦状态
    - "发布"按钮仍可点击（未禁用）

- 输入内容超过 10000 字符，前端阻止继续输入并提示
  - 前置条件：用户已登录，正在输入笔记
  - 操作步骤：
    1. 在输入框中输入 10001 字符
  - 预期结果：
    - 输入框底部字数提示变红（如"10001/10000"）
    - 输入框上方或附近显示警告提示："已达字数上限"
    - 无法继续输入（或继续输入的字符被自动截断）

- 未登录用户访问主界面，跳转到登录页
  - 前置条件：用户未登录（无有效 Session）
  - 操作步骤：
    1. 直接访问主界面 URL（如 `/`）
  - 预期结果：
    - 页面自动跳转至登录页（如 `/login`）
    - 登录页正常展示登录表单

- 网络异常时点击发布，保留已输入内容并提示错误
  - 前置条件：用户已登录，输入框中有内容"测试网络异常"
  - 操作步骤：
    1. 断开网络连接（模拟）
    2. 点击"发布"按钮
  - 预期结果：
    - 界面显示错误提示："网络连接失败，请稍后重试"
    - 输入框中的内容"测试网络异常"仍然保留
    - 用户可重新点击发布按钮

### API 测试场景

#### 正常场景

- 有效内容和已登录用户，笔记创建成功
  - 操作步骤：
    1. 用户已登录（Session 有效，携带 Cookie）
    2. 发送 POST /api/memos，Body：`{ "content": "今天学到的新概念" }`
  - 预期结果：
    - 接口返回 201
    - 响应体格式：`{ "data": { "id": "uuid", "content": "今天学到的新概念", "userId": "uuid", "createdAt": "ISO时间", "updatedAt": "ISO时间", "tags": [], "images": [] }, "message": "创建成功" }`
    - 数据库 memos 表中新增一条记录，content 字段值为"今天学到的新概念"

- content 为 10000 字符（边界值），创建成功
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body content 字段为 10000 个字符
  - 预期结果：
    - 接口返回 201
    - 响应体 data.content 长度为 10000
    - 数据库成功存储完整内容

- content 为 1 字符（最小边界值），创建成功
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{ "content": "a" }`
  - 预期结果：
    - 接口返回 201
    - 响应体 data.content 值为 "a"

#### 异常场景

- 未登录时访问创建接口，返回 401
  - 操作步骤：
    1. 不携带 Session cookie（或 cookie 无效）
    2. 发送 POST /api/memos，Body：`{ "content": "测试" }`
  - 预期结果：
    - 接口返回 401
    - 响应体格式：`{ "data": null, "error": "Unauthorized", "message": "请先登录" }`

- content 为空字符串，返回 400
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{ "content": "" }`
  - 预期结果：
    - 接口返回 400
    - 响应体格式：`{ "data": null, "error": "VALIDATION_ERROR", "message": "请求参数不合法" }` 或类似错误提示

- content 字段缺失，返回 400
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{}`（不包含 content 字段）
  - 预期结果：
    - 接口返回 400
    - 响应体包含错误信息，提示缺少必填字段

- content 超过 10000 字符，返回 400
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body content 字段为 10001 个字符
  - 预期结果：
    - 接口返回 400
    - 响应体格式：`{ "data": null, "error": "VALIDATION_ERROR", "message": "请求参数不合法" }`

- 数据库异常时，返回 500 且不暴露内部错误
  - 操作步骤：
    1. 用户已登录
    2. 模拟数据库故障（如数据库文件锁定或连接断开）
    3. 发送 POST /api/memos，Body：`{ "content": "测试" }`
  - 预期结果：
    - 接口返回 500
    - 响应体格式：`{ "data": null, "error": "INTERNAL_ERROR", "message": "服务器内部错误" }`（生产环境）
    - 响应体不包含数据库详细错误信息（如表名、字段名、SQL 语句）

## 创建带标签的笔记

### UI 测试场景

#### 正常场景

- 选择已有标签并发布，笔记显示该标签
  - 前置条件：用户已登录，已有标签"#阅读"
  - 操作步骤：
    1. 在输入框中输入"《原则》第三章笔记"
    2. 点击"#标签"按钮或标签选择入口
    3. 从弹出的标签列表中选择"#阅读"
    4. 弹层关闭，输入框上方显示"#阅读"
    5. 点击"发布"按钮
  - 预期结果：
    - 笔记成功发布
    - 笔记卡片上显示"#阅读"标签
    - 切换到"有标签"分类，该笔记可见

- 输入不存在的标签，系统自动创建新标签
  - 前置条件：用户已登录，不存在标签"#新标签"
  - 操作步骤：
    1. 在输入框中输入"测试新标签功能"
    2. 打开标签选择弹层
    3. 在标签输入框中输入"新标签"
    4. 点击"添加"按钮
    5. 弹层关闭，输入框上方显示"#新标签"
    6. 点击"发布"按钮
  - 预期结果：
    - 笔记成功发布
    - 笔记卡片上显示"#新标签"
    - 进入"全部标签"页面，列表中出现"#新标签"条目，笔记数为 1

- 为笔记添加多个标签，笔记在所有标签筛选下均可见
  - 前置条件：用户已登录，已有标签"#工作"和"#灵感"
  - 操作步骤：
    1. 在输入框中输入"新产品创意"
    2. 选择标签"#工作"和"#灵感"
    3. 输入框上方显示"#工作 #灵感"
    4. 点击"发布"按钮
  - 预期结果：
    - 笔记成功发布
    - 笔记卡片上同时显示"#工作"和"#灵感"两个标签
    - 进入"全部标签"页面，"#工作"和"#灵感"的笔记数各 +1
    - 分别点击"#工作"和"#灵感"标签进入对应笔记列表，该笔记均可见

#### 异常场景

- 标签名称包含特殊字符，前端实时过滤
  - 前置条件：用户已登录，正在添加新标签
  - 操作步骤：
    1. 打开标签选择弹层
    2. 在标签输入框中输入"工作@home"
  - 预期结果：
    - 输入框中仅显示"工作home"（特殊字符 @ 被自动过滤）
    - 或显示提示："标签仅支持字母、数字、下划线"

- 标签名称超过 20 字符，前端自动截断
  - 前置条件：用户已登录，正在添加新标签
  - 操作步骤：
    1. 打开标签选择弹层
    2. 在标签输入框中输入 21 个字符的标签名
  - 预期结果：
    - 输入框中仅显示前 20 个字符
    - 或无法继续输入第 21 个字符

- 标签名称少于 2 字符，无法添加
  - 前置条件：用户已登录，正在添加新标签
  - 操作步骤：
    1. 打开标签选择弹层
    2. 在标签输入框中输入单个字符"a"
    3. 点击"添加"按钮
  - 预期结果：
    - 显示错误提示："标签名称至少 2 个字符"
    - 标签未添加到笔记

### API 测试场景

#### 正常场景

- 创建笔记时传入已存在的标签 ID，关联成功
  - 操作步骤：
    1. 用户已登录，已有标签"#阅读"（id: "tag-uuid-1"）
    2. 发送 POST /api/memos，Body：`{ "content": "测试", "tagNames": ["阅读"] }`
  - 预期结果：
    - 接口返回 201
    - 响应体 data.tags 数组包含 `{ "id": "tag-uuid-1", "name": "阅读" }`
    - 数据库 memo_tags 表中新增一条记录，memoId 和 tagId 正确关联

- 创建笔记时传入不存在的标签名，后端自动创建
  - 操作步骤：
    1. 用户已登录，不存在标签"#新标签"
    2. 发送 POST /api/memos，Body：`{ "content": "测试", "tagNames": ["新标签"] }`
  - 预期结果：
    - 接口返回 201
    - 响应体 data.tags 数组包含新创建的标签对象
    - 数据库 tags 表中新增一条记录，name 为"新标签"，userId 为当前用户
    - 数据库 memo_tags 表中新增关联记录

- 创建笔记时传入多个标签，全部关联成功
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{ "content": "测试", "tagNames": ["工作", "灵感", "阅读"] }`
  - 预期结果：
    - 接口返回 201
    - 响应体 data.tags 数组包含 3 个标签对象
    - 数据库 memo_tags 表中新增 3 条记录

#### 异常场景

- tagNames 数组中包含非法字符的标签名，返回 400
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{ "content": "测试", "tagNames": ["工作@home"] }`
  - 预期结果：
    - 接口返回 400
    - 响应体格式：`{ "data": null, "error": "VALIDATION_ERROR", "message": "请求参数不合法" }`

- tagNames 数组中包含长度超过 20 字符的标签名，返回 400
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{ "content": "测试", "tagNames": ["一个超过二十个字符的标签名称"] }`
  - 预期结果：
    - 接口返回 400

- tagNames 数组中包含长度少于 2 字符的标签名，返回 400
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{ "content": "测试", "tagNames": ["a"] }`
  - 预期结果：
    - 接口返回 400

- tagNames 数组超过 50 项，返回 400
  - 操作步骤：
    1. 用户已登录
    2. 发送 POST /api/memos，Body：`{ "content": "测试", "tagNames": ["标签1", "标签2", ..., "标签51"] }`（51 个标签）
  - 预期结果：
    - 接口返回 400

## 创建带图片的笔记

### UI 测试场景

#### 正常场景

- 选择本地图片（1.5MB JPG），预览显示正常
  - 前置条件：用户已登录，输入框中有内容
  - 操作步骤：
    1. 在输入框中输入"今天的美食"
    2. 点击"📷图片"按钮
    3. 从本地相册选择一张 1.5MB 的 JPG 图片
  - 预期结果：
    - 本地图片选择器关闭
    - 输入框下方显示图片缩略图
    - 缩略图右上角显示删除按钮（× 图标）
    - 图片未上传时可能显示上传进度条

- 添加图片后点击删除按钮，图片从笔记中移除
  - 前置条件：用户已添加一张图片，图片缩略图显示在输入框下方
  - 操作步骤：
    1. 点击图片缩略图右上角的删除按钮（× 图标）
  - 预期结果：
    - 图片缩略图从界面消失
    - 点击"发布"后，笔记不包含该图片

- 添加多张图片（最多 9 张），全部显示在笔记中
  - 前置条件：用户已登录
  - 操作步骤：
    1. 在输入框中输入"今天的相册"
    2. 点击"📷图片"按钮，连续选择 9 张图片
    3. 点击"发布"按钮
  - 预期结果：
    - 输入框下方显示 9 张图片缩略图
    - 笔记成功发布
    - 笔记卡片显示 9 张图片缩略图

#### 异常场景

- 选择第 10 张图片时，系统提示"最多添加 9 张图片"
  - 前置条件：用户已添加 9 张图片
  - 操作步骤：
    1. 点击"📷图片"按钮
    2. 尝试选择第 10 张图片
  - 预期结果：
    - 显示提示："最多添加 9 张图片"
    - 第 10 张图片未添加

- 上传的图片超过 5MB，系统提示错误
  - 前置条件：用户已登录
  - 操作步骤：
    1. 点击"📷图片"按钮
    2. 选择一张 6MB 的图片
  - 预期结果：
    - 显示提示："图片大小不能超过 5MB"
    - 图片未添加到笔记

- 上传不支持的图片格式（如 PSD、TIFF），系统提示错误
  - 前置条件：用户已登录
  - 操作步骤：
    1. 点击"📷图片"按钮
    2. 选择一张 PSD 格式的文件
  - 预期结果：
    - 显示提示："仅支持 JPG、PNG、GIF 格式"
    - 图片未添加到笔记
