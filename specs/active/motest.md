# Feature Spec: 连通性探测接口（Health Check）

**作者**: momo
**日期**: 2026-03-01
**状态**: 已批准
**关联 Issue**: —

---

## 1. 背景

在开发和运维阶段，需要一个极简的 HTTP 接口来验证服务是否正常运行、网络可达。
该接口不涉及任何业务逻辑，仅作为"心跳"探针使用，供脚本、监控系统或手动 curl 调用。

## 2. 用户故事

```
作为开发者或运维人员，
我希望能通过 HTTP GET 请求判断服务是否可达，
以便在部署、排障时快速确认服务状态，无需依赖业务接口。
```

## 3. 功能描述

- 提供一个只读 HTTP 端点 `GET /api/health`
- 接口无需鉴权，任何来源均可访问
- 服务正常时返回 HTTP 200 及固定 JSON 响应体
- 接口本身不查询数据库、不依赖外部服务

## 4. 数据模型

**不涉及数据库操作。**
接口返回固定内容，无需读写任何数据表。

## 5. API 设计

### 端点

```
GET /api/health
```

### 请求

无请求参数，无请求体，无需 Authorization 头。

### 响应（成功，HTTP 200）

```json
{
  "status": "ok"
}
```

### 错误码

| 状态码 | 场景 |
|--------|------|
| 200 | 服务正常，响应体 `{ "status": "ok" }` |
| 500 | 服务内部异常（Route Handler 抛出未捕获错误） |

> 正常情况下只应出现 200。若出现 500，说明部署环境存在问题，需排查。

## 6. 边界条件

| 场景 | 处理方式 |
|------|----------|
| POST / PUT 等非 GET 方法 | Next.js 默认返回 405 Method Not Allowed |
| 服务进程崩溃 | TCP 连接拒绝，非接口层面问题 |
| 接口收到请求体 | 忽略，不解析 |
| 并发大量请求 | 接口无状态，无锁，天然支持并发 |

## 7. 不包含

- **不**返回版本号、构建信息、环境变量等敏感信息
- **不**检查数据库连接状态（避免引入外部依赖）
- **不**需要鉴权或 API Key
- **不**做限流（MVP 阶段）

## 8. 验证标准

### 功能验证

```bash
# 启动开发服务器后执行
curl -i http://localhost:3000/api/health
```

预期结果：
- HTTP 状态码：`200 OK`
- 响应体：`{"status":"ok"}`
- Content-Type：`application/json`

### 技术验证

- [ ] Route Handler 文件位于 `apps/flomo/app/api/health/route.ts`
- [ ] 仅导出 `GET` 方法
- [ ] 响应通过 `NextResponse.json` 返回，不硬编码字符串
- [ ] `npm run build` 无编译错误

### 性能验证

- 接口响应时间 < 50ms（本地环境）
- 无数据库查询，无 I/O 操作
